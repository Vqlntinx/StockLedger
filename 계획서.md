# 소프트웨어 계획서
# 프로젝트 명 : StockLedger ~주식 거래 관리 시스템~
## 1. 서론
    1.1 목적
    본 문서는 Python 기반으로 개발되는 StockLedger 프로젝트의 소프트웨어 프로젝트 관리 계획서(SMP)를 기술한다.
    계획서는 IEEE 1058-1998 표준에 따라 프로젝트의 목적, 범위, 조직, 일정, 관리 프로세스, 기술적 프로세스 등을 명확하게 정의하여 개발의 일관성과 품질을 보장하는 것을 목표로 한다.

    1.2 프로젝트 개요 및 범위
    StockLedger는 개인 투자자가 실행한 주식 매수·매도 거래 내역을 기록하고, 종목별 평균 매입단가, 실현손익, 보유 수량 및 총 매입금액을 자동 계산해주는 가벼운 손익 관리 시스템이다.

    1.3 목표
    - 주식 거래 기록을 체계적으로 저장하고 관리
    - 평균 단가 기반 실현손익 계산
    - 객체지향 설계 원리 및 SOLID 준수
    - 디자인 패턴을 활용한 확장성 보장
    - 단위 테스트를 활용한 신뢰성 확보

    1.4 정의 및 약어
    |용어/약어|	설명|
    |P/L	|Profit/Loss (손익)|
    |CLI	|Command-Line Interface|
    |Repository Pattern	|데이터 저장 방식 추상화 설계 패턴|
    |Strategy Pattern	|알고리즘 교체가 가능한 설계 패턴|

    1.5 참고 문서
    -IEEE Std 1058-1998, Software Project Management Plans
    -IEEE Std 830-1998, Software Requirements Specification
    -Somerville, Software Engineering, 10th Edition
    -Python 공식 문서

## 2. 프로젝트 조직
    1인 개발으로 진행되었기에 내용은 생략하나, 프로젝트 관리, 요구사항 분석, 아키텍트 설계, 개발, 테스트, 문서 등을 전부 혼자 진행하였음.

## 3. 관리 프로세스 계획
    3.1 시작 계획
    비용 및 노력 추정치 : 약 20시간, 코드 규모 : 400 ~ 550 LOC
    리소스 계획 : Python 3 개발 환경, PC, 설계 도구(draw.io), Github
    
    3.2 작업 구조 계획
    Level 1 - 전체 작업 구조 : 요구사항 분석 - 설계 - 구현 - 테스트 - 문서화
    
    3.3 일정 계획
    요구사항 분석 (2일) - 설계 (3일) - 구현 (3일) - 테스트(1일) - 문서화(2일)

    3.4 위험 관리 계획
    계산 로직 오류 : Strategy 패턴 사용 + 단위 테스트 강화
    파일 I/O 오류 : 예외처리 강화, 백업 파일 생성
    잘못된 사용자 입력 : CLI에서 입력 검증 추가
    일정 지연 : 작업 구조 계획 기반 일정 관리

## 4. 기술 프로세스 계획
    4.1 개발 프로세스 모델 : '폭포수 모델'을 기반으로 진행, 소규모 프로젝트 특성상 설계, 구현, 테스트 간에 부분 반복 허용.

    4.2 방법론
    개발 언어 : Python
    아키텍처 스타일 : Layered Architecture + MVC 구조
    설계 패턴 : Repository Pattern (데이터 저장 추상화), Strategy Pattern (알고리즘 교체)
    설계 기법 : UML
    테스트 : 유닛테스트 기반
