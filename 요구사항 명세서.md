# 소프트웨어 요구사항 명세서
# Based on IEEE 830-1998.
# Project name : StockLedger
## 1. 서론
    1.1 목적
    본 문서는 Python 기반 CLI 프로그램인 StockLedger의 소프트웨어 요구사항 명세서(SRS)로서, 시스템의 기능적/비기능적 요구사항을 IEEE 830 표준에 따라 명확하게 기술한다.

    1.2 범위
    StockLedger는 개인 투자자가 실행한 주식 거래 내역을 기록하고, 보유 종목 현황(수량, 총 매입금액, 평균 매입단가), 실현손익, 종목별 손익 분석을 CLI 환경에서 제공하는 경량 소프트웨어이다.
    
    1.3 참조 문서
    IEEE Std 830-1998, Recommended Practice for Software Requirements Specifications
    IEEE Std 1058-1998, Software Project Management Plans

## 2. 전반적인 설명
    2.1 제품 관점
    독립 실행형 로컬 프로그램이며, 외부 시스템과 연동하지 않는다.
    UI 레이어 : CLI
    서비스 레이어 : 손익 계산 처리
    저장소 레이어 : JSON 활용 파일 저장

    2.2 제품 기능 요약
    1. 거래 기록 추가
    2. 보유 종목 현황 조회
    3. 종목별 실현손익 조회
    4. 전체 실현손익 조회
    5. 데이터 파일 저장/로드

    2.3 사용자 특성
    주식 투자자, 기본적 CLI 사용가능, 금융 초보자 Level.

## 3. 구체적 요구사항
    3.1 기능적 요구사항 (Functional Requirements, FR 약칭으로 처리함)
    1. 거래 추가 : 사용자는 BUY 또는 SELL 거래를 입력할 수 있다.
    2. 보유 현황 조회 : 시스템은 종목별 보유 수량, 평균 매입단가, 총 매입금액을 게산해 출력해야한다.
    3. 종목별 실현손익 조회 : 매도 시 평균 매입단가 기준으로 계산된 손익을 종목별로 출력한다.
    4. 전체 실현손익 조회 : 모든 종목의 실현손익을 합산하여 출력한다.
    5. 데이터 저장 및 로드 : 거래 데이터는 JSON 형태로 저장되고, 프로그램 실행 시 자동으로 로드한다.

    3.2 비기능적 요구사항
    1. 신뢰성 : 파일 저장 실패/손상 시 오류 메시지 출력
    2. 사용성 : 초보자도 사용 가능한 간단한 CLI 명령어 구성
    3. 유지보수성 : 디자인 패턴을 활용한 저장방식 교체 등 용이
    4. 확장성 : 향후 DB 기반 저장소로 확장 가능하도록 계층 분리

    3.3 시스템 속성
    1. 안정성 : 정상 종료 전 파일 자동 저장
    2. 보안성 : 로컬에서만 실행
    3. 오류내구성 : 파일 손상 시 신규 파일 생성
